YOUR ACTION ITEMS

1️⃣  GENERATE SECRETS (Run Now)
   
   Open terminal and run:
   
   $ openssl rand -hex 32
   Save the output → This is your SECRET_KEY
   
   $ openssl rand -hex 16
   Save the output → Optional (Railway sets this)

2️⃣  GET STRIPE KEYS (If using Stripe)
   
   Go to: https://dashboard.stripe.com/apikeys
   Copy: Live Secret Key (sk_live_...)
   Copy: Live Webhook Secret (whsec_...)

3️⃣  PUSH TO GITHUB (Make sure code is committed)
   
   $ git status
   $ git add .
   $ git commit -m "Ready for deployment"
   $ git push origin main

4️⃣  GO TO RAILWAY AND DEPLOY
   
   Open: https://railway.app/dashboard


RAILWAY DEPLOYMENT STEPS (15 MINUTES)
══════════════════════════════════════════════════════════════════

Step 1: Sign Up (2 min)
────────────────────────

  Go to: https://railway.app
  Click: "Sign up"
  Use: GitHub login (recommended)
  Then: Authorize Railway


Step 2: Create Project (1 min)
──────────────────────────────

  Go to: https://railway.app/dashboard
  Click: "+ New Project"
  Select: "Deploy from GitHub"


Step 3: Connect GitHub (2 min)
──────────────────────────────

  Click: "Configure GitHub App"
  Select: Your GitHub account
  Choose: "Only select repositories"
  Search: "saas_platform"
  Click: "Install"


Step 4: Deploy Repository (1 min)
──────────────────────────────────

  Click: "Deploy from GitHub"
  Select: "saas_platform"
  Branch: "main"
  Click: "Deploy"

  (Railway now reads railway.json and creates services)


Step 5: Wait for Auto-Setup (3 min)
────────────────────────────────────

  Watch the dashboard as Railway:
  ✓ Creates PostgreSQL database
  ✓ Builds backend image
  ✓ Builds frontend image
  ✓ Starts all services
  
  All 3 services should show "Running"


Step 6: Add Stripe Keys (2 min)
────────────────────────────────

  Click: "backend" service
  Go to: "Variables" tab
  Add:
    SECRET_KEY = <your generated secret>
    STRIPE_SECRET_KEY = sk_live_xxxxx
    STRIPE_WEBHOOK_SECRET = whsec_xxxxx
  
  Click: "Save"

  (Railway auto-redeploys with new env vars)


Step 7: Get Your URLs (1 min)
──────────────────────────────

  Click: "frontend" service → "Deployments"
  Copy: https://saas-frontend-prod-xxxxx.up.railway.app
  
  Click: "backend" service → "Deployments"
  Copy: https://saas-backend-prod-xxxxx.up.railway.app
  
  Save these! They're your live URLs!


Step 8: Test Everything (2 min)
────────────────────────────────

  Test API:
    curl https://saas-backend-prod-xxxxx.up.railway.app/docs
  
  Test Frontend:
    Open https://saas-frontend-prod-xxxxx.up.railway.app
  
  Test Registration:
    - Go to frontend
    - Click "Register"
    - Create account
    - Should work!


TOTAL TIME: ~15 minutes from start to live! ⚡

TROUBLESHOOTING
══════════════════════════════════════════════════════════════════

Problem: Services stuck on "Building"
Solution: Check logs, verify Dockerfiles work locally

Problem: Backend can't connect to database
Solution: Check DATABASE_URL env var, verify db is running

Problem: Frontend can't reach backend
Solution: Check VITE_API_URL is set to your backend URL

Problem: Stripe payment fails
Solution: Verify you're using LIVE keys (sk_live_), not test keys

For more help:
  - Railway Docs: https://docs.railway.app
  - Discord: https://discord.gg/railway
